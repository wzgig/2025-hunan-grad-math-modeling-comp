# 二、问题2：建模与算法设计
## 2.1 问题界定与目标

**系统**：大厅**2 个测试台**（在制品 WIP≤2），**4 个专用工位** A/B/C/E（各 1 套设备）。装置运入/运出各 $0.5$ h。A/B/C 可在**同一台装置上并行**测试，**E 需在 A/B/C 全部通过后**进行。每日单班，**班长 12 h**；任何工序一旦中断（含班次截止、设备故障）须**从头重测**；同一道工序**两次未通过**则该装置退出。

**目标**（双目标）：

$$
\min \ \mathbb{E}[T]\quad\text{与}\quad \min \ P_L
$$

并输出：完成天数 $T$、通过数量 $S$、总漏判 $P_L$、总误判 $P_W$、有效工作时间比 $YXB_j\,(j\in\{A,B,C,E\})$。

## 2.2 符号体系（集合、变量、参数）

**集合**
$\mathcal{I}=\{1,\dots,100\}$：装置；$\mathcal{J}=\{A,B,C,E\}$：工位；$\mathcal{J}_{sub}=\{A,B,C\}$：子系统工位；$\mathcal{K}=\{1,2\}$：测试台。

**状态变量**

* $\theta_{i,j}\in\{0,1\}$：装置 $i$ 的子系统 $j\in\{A,B,C,D\}$ 是否**真实有问题**；
* “进入 E 时”的**综合真问题**状态

  $$
  \theta_{i,E}=\theta_{i,A}^{\text{残}}\lor \theta_{i,B}^{\text{残}}\lor \theta_{i,C}^{\text{残}}\lor \theta_{i,D},
  $$

  其中 $\theta_{i,j}^{\text{残}}$ 表示 $i$ 在通过工位 $j$ 后**仍然存在**的问题（被漏判未拦截）；
* $\pi_{i,j}\in\{0,1\}$：装置 $i$ 是否最终通过工位 $j$；
* $\tau_{i,j}\in\{0,1,2\}$：装置 $i$ 在工位 $j$ 的尝试次数；
* $x_{i,j}(t)\in\{0,1\}$：时刻 $t$ 装置 $i$ 是否在工位 $j$ 测试；
* $y_{i,k}(t)\in\{0,1\}$：时刻 $t$ 装置 $i$ 是否占用测试台 $k$；
* $L_j(t)$：工位 $j$ 设备累计使用时长；$F_j(t)\in\{0,1\}$：是否故障。

**参数**（题设 + 问题1口径）

* 固有问题率：$p_A=2.5\%,p_B=3\%,p_C=2\%,p_D=0.1\%$；
* 差错率：$e_A=3\%,e_B=4\%,e_C=2\%,e_E=2\%$，误/漏各占 50%；
* 由此得**子系统漏/误判率**：$\alpha_j=\frac{0.5e_j}{p_j},\ \beta_j=\frac{0.5e_j}{1-p_j},\ j\in\{A,B,C\}$；
* 标准工时：$T_A=2.5\text{h},T_B=2\text{h},T_C=2.5\text{h},T_E=3\text{h}$；
* 调试（首次启用/更换后）：$(30,20,20,40)$ min；运输：$T^{in}=T^{out}=0.5\text{h}$；
* 故障（分段累计）：$0\!\sim\!120$h：A/B/C/E=3/4/2/3%，$120\!\sim\!240$h：5/7/6/5%，$\ge240$h 或故障则**必须更换**，$\ge120$h 可**预防更换**。

## 2.3 约束体系（修正版）

### 2.3.1 容量与占用

$$
\sum_{i}y_{i,k}(t)\le1,\ \forall k;\quad 
\sum_{i}x_{i,j}(t)\le1,\ \forall j;\quad 
x_{i,j}(t)\le \sum_{k}y_{i,k}(t),\ \forall i,j.
$$

（同一时刻最多两台装置在场；每个工位单线程；测试必须在测试台上进行。）

### 2.3.2 流程与尝试上限

$$
\sum_{t'\le t}x_{i,E}(t') \le \min_{j\in\mathcal{J}_{sub}}\pi_{i,j},\ \forall i,t;\quad 
\tau_{i,j}\le2,
$$

若 $\tau_{i,j}=2\land \pi_{i,j}=0\Rightarrow i$ 退出（对 E 亦同）。

### 2.3.3 **班次窗口可行性（关键修正）**

在班次 $[t_s,t_e]$ 内，**禁止**启动任一会跨班的工序：

$$
x_{i,j}(t)=1\Rightarrow t+T_j^{\text{rem}}(i,j)\le t_e,
$$

其中 $T_j^{\text{rem}}(i,j)$ 为该工序**从头开始**所需时长（题设“不可接续”）。

> 该规则按**绝对时间轴**判断，不对不同工位的时长求和，因此天然支持 A/B/C 并行；若剩余时间不足则延后到下一班，避免“启动即作废”的浪费。

## 2.4 随机过程与质量判定（统一口径）

### 2.4.1 子系统工位（A/B/C）

对 $j\in\{A,B,C\}$：

$$
\mathbb{P}(\text{通过}_j\mid \theta_{i,j}=1)=\alpha_j,\quad 
\mathbb{P}(\text{通过}_j\mid \theta_{i,j}=0)=1-\beta_j.
$$

“**残留后验**”（进入 E 前）：

$$
p_j^{\text{res}}=\mathbb{P}(\theta_{i,j}=1\mid \text{通过}_j)
=\frac{p_j\alpha_j}{p_j\alpha_j+(1-p_j)(1-\beta_j)}.
$$

### 2.4.2 综合工位（E）的**联合先验与差错分配**

进入 E 时“存在任一真问题”的基率

$$
p_{\text{any}}=1-\prod_{j\in\{A,B,C\}}\bigl(1-p^{\text{res}}_j\bigr)\cdot(1-p_D).
$$

据“误/漏各占 $e_E$ 的一半”：

$$
\alpha_E=\frac{0.5\,e_E}{p_{\text{any}}},\qquad 
\beta_E=\frac{0.5\,e_E}{1-p_{\text{any}}}.
$$

E 的判别：

$$
\mathbb{P}(\text{通过}_E\mid \theta_{i,E}=1)=\alpha_E,\quad 
\mathbb{P}(\text{通过}_E\mid \theta_{i,E}=0)=1-\beta_E.
$$

**全系统漏判与误判**

* 单装置“**最终漏判**”（带病通过）的事件为：
  $\bigwedge_{j\in\{A,B,C\}} \text{通过}_j$ 且 $\theta_{i,E}=1$ 且 **E 通过**。其概率（对给定 $i$）为

  $$
  P_{L,i}= \Bigl[\prod_{j} \bigl(p_j\alpha_j+(1-p_j)(1-\beta_j)\bigr)\Bigr]\cdot p_{\text{any}}\cdot \alpha_E.
  $$
* “**最终误判**”（无病未过）由任一工位误判或 E 误判触发且**经两次失败退出**，在仿真统计中按频率估计 $P_W$；解析上可分解但表达较长，论文中给出**仿真估计**口径并说明置信区间。

## 2.5 设备故障与更换（时间化成本口径）

### 2.5.1 分段风险与失效时间

令阶段累计故障率为 $(r^{(1)}_j,r^{(2)}_j)$。对使用时长 $u\in(0,120]$：

$$
\mathbb{P}(\text{故障于 }u)=\frac{r^{(1)}_j}{120},\quad 
(120,240]:\ \frac{r^{(2)}_j-r^{(1)}_j}{120}.
$$

达到 240h 或发生故障**强制更换**；更换后计一次 $T^{\text{setup}}_j$ 并将 $L_j$ 归零。

### 2.5.2 预防性更换的期望代价

$$
C_{\text{replace}}=T^{\text{setup}}_j,\qquad 
C_{\text{failure}}=T^{\text{setup}}_j+\mathbb{E}[\text{中断损失}]+T^{\text{test}}_j.
$$

中断损失按“**作废本次工序已耗时的 50% 期望**”近似：$\mathbb{E}[\text{中断损失}]\approx 0.5\,T^{\text{test}}_j$。
**决策规则（班次边界评估）**：若 $L_j\ge120$ 且

$$
\Delta \text{Risk}=\frac{r^{(2)}_j-r^{(1)}_j}{120}\cdot C_{\text{failure}} > C_{\text{replace}},
$$

则在班次开始进行预防更换；$L_j\ge240$ 必换。

## 2.6 目标与指标计算

* **主目标**：$\min \mathbb{E}[T]$（完成 100 台所需天数的期望）。
* **次目标**：$\min P_L$，其中

$$
P_L=\frac{1}{S}\sum_{i\in\mathcal{I}_{\text{pass}}}\mathbb{1}[\theta_{i,E}=1],\quad 
S=|\mathcal{I}_{\text{pass}}|.
$$

* **总误判**：$P_W=\dfrac{\#\{\text{实际无问题但最终未通过}\}}{100-S}$（仿真估计）。
* **有效工作时间比**：$YXB_j=\dfrac{\text{工位 }j\text{ 的“纯测试占用时长”}}{12\times \text{总班次数}}$。

## 2.7 求解算法（DES + 启发式派工）

### 2.7.1 事件驱动框架

事件集 $\mathcal{E}=\{$入场完成/运入完成、测试开始、测试完成、设备故障、设备更换完成、班次结束$\}$。使用**小根堆事件队列**按事件时间推进全局时钟。每到**班次边界**：终止未完成工序（作废），统计浪费，并在下一班重排。

### 2.7.2 派工与“不开工规则”

* **并行分配**：同一时刻最多 2 台装置在场；对每台装置并发派发 A/B/C（视工位空闲）。
* **班次窗口可行性**（强约束）：若 $t_{\text{now}}+T_j^{\text{test}}>t_{\text{shift\_end}}$，则**禁止**启动工位 $j$（避免“起而复废”）。
* **派工优先级（可复现）**：

  $$
  \text{Priority}_i=3\cdot \text{Progress}_i+2\cdot \widetilde{\text{Wait}}_i+1\cdot \widetilde{\text{SPT}}_i^{-1},
  $$

  其中 $\text{Progress}_i=\tfrac{1}{3}\sum_{j\in\{A,B,C\}}\pi_{i,j}$，$\widetilde{\text{Wait}}$ 与 $\widetilde{\text{SPT}}$ 为在当前队列内归一化的等待时长与剩余加工时间；权重 $(3,2,1)$ 源自针对标量化目标 $w_T\mathbb{E}[T]+w_LP_L$ 的**网格校准**（附实验流程，便于复现）。

### 2.7.3 E 工位与重测

* E 的启动条件：$\pi_{i,A}=\pi_{i,B}=\pi_{i,C}=1$。
* 若 E 检出问题（含误判）则**重测 E**；连续两次未通过则退出（与题面一致）。

### 2.7.4 设备维护决策

* 每班开班先检查 $L_j$ 并按 2.5.2 执行更换；
* 故障发生即刻作废当前工序并入队更换（计 $T^{\text{setup}}_j$），更换完成后该工位恢复可用。

## 2.8 复杂度与再现性

* **时间复杂度**：事件驱动 $O(M\log M)$，$M$ 为总事件数（与工序数、重测与故障次数线性相关）；
* **再现性**：给出随机数种子与权重 $(3,2,1)$、维护阈值（$\ge120$h 评估，更换判据如 2.5.2），可完全复现。

    end

            obj.isOccupied = false;
            obj.currentDevice = 0;
            obj.equipment = Equipment(name, params);
            obj.testStartTime = 0;
            obj.totalTestTime = 0;
            obj.totalIdleTime = 0;
        end
        
        function available = isAvailable(obj)
            available = ~obj.isOccupied && ~obj.equipment.isFailed;
        end
        
        function startTest(obj, deviceId, currentTime)
            obj.isOccupied = true;
            obj.currentDevice = deviceId;
            obj.testStartTime = currentTime;
        end
        
        function completeTest(obj, currentTime)
            testDuration = currentTime - obj.testStartTime;
            obj.totalTestTime = obj.totalTestTime + testDuration;
            obj.equipment.addUsageTime(testDuration);
            obj.isOccupied = false;
            obj.currentDevice = 0;
        end
        
        function efficiency = getEfficiency(obj, totalTime)
            efficiency = obj.totalTestTime / totalTime;
        end
    end
end

% Classes/EventQueue.m
classdef EventQueue < handle
    %EVENTQUEUE 事件优先队列
    
    properties (Access = private)
        events  % 事件数组
        count   % 事件数量
    end
    
    methods
        function obj = EventQueue()
            obj.events = Event.empty();
            obj.count = 0;
        end
        
        function addEvent(obj, event)
            %添加事件（保持时间顺序）
            obj.events(end+1) = event;
            obj.count = obj.count + 1;
            
            % 冒泡上浮，维护最小堆
            idx = obj.count;
            while idx > 1
                parentIdx = floor(idx/2);
                if obj.events(idx).time < obj.events(parentIdx).time
                    % 交换
                    temp = obj.events(idx);
                    obj.events(idx) = obj.events(parentIdx);
                    obj.events(parentIdx) = temp;
                    idx = parentIdx;
                else
                    break;
                end
            end
        end
        
        function event = getNext(obj)
            %获取最早的事件
            if obj.isEmpty()
                event = [];
                return;
            end
            
            event = obj.events(1);
            
            % 将最后一个元素移到顶部
            obj.events(1) = obj.events(obj.count);
            obj.events(obj.count) = [];
            obj.count = obj.count - 1;
            
            % 下沉调整
            obj.heapifyDown(1);
        end
        
        function empty = isEmpty(obj)
            empty = (obj.count == 0);
        end
        
        function heapifyDown(obj, idx)
            %堆调整
            while true
                leftChild = 2 * idx;
                rightChild = 2 * idx + 1;
                smallest = idx;
                
                if leftChild <= obj.count && ...
                   obj.events(leftChild).time < obj.events(smallest).time
                    smallest = leftChild;
                end
                
                if rightChild <= obj.count && ...
                   obj.events(rightChild).time < obj.events(smallest).time
                    smallest = rightChild;
                end
                
                if smallest ~= idx
                    temp = obj.events(idx);
                    obj.events(idx) = obj.events(smallest);
                    obj.events(smallest) = temp;
                    idx = smallest;
                else
                    break;
                end
            end
        end
    end
end
